PBO Manager
PBO Manager is similar to known Amalfi's MakePBO and UnPBO or DePBO or StuffPBO and its task is to pack and unpack already packed PBO files while optionaly using compression. The big advance is the possibility to extract only one or more files from already packed PBO without a necessary unpacking all packed files.
I wanted to build up a quickier and less memory consuming tool to handle PBO files with better compression what turned out as a hard task (because of BIStudio's hybrid LZ77 algorithm) but I think that the conclusion is satisfactory. Using the PBO Manager you can get best compression ratio from the tools named above including OFP engine itself and quick handling.
In advance the PBO Manager has an ability to decode OFP BINary files and this function is even possible to use directly from OFP MDE through the local menu of the right projects panel.
It's few months (maybe year) ago I read a discussion on ofp.info forum where contributors (addon makers) discussed the problem of code stealing. The conclussion was that the BIN files should stay binarized and nobody should decode it. It was probably the reason why Amalfi removed the decoder from his UnPBO program (by the version 1.3 it was implemented). I'm sure this misgiving has its foundation but I'm also the protagonist of the open-source because of for me personally the best way how to learn something is to read code samples. It's also my opinion that the computer world globaly tends to the open-source and the best example is the MSIL(Microsoft Intermediate Language) of Microsoft which came with NET.Framework. Since this time each code using this language is actually an open-source even the code of Microsoft products themselfs. Well... at this moment I ask myself... when Microsoft can, why OFP community couldn't ? :o)) With this short explanation of my opinion I would like to end up any contingent discussion about this topic and at the end I would like to remind to all... decode, read and learn but DON'T use any unauthorized code. The OFP community isn't so big that nobody could reveal you.
