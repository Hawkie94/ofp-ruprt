Syntax highlighter
This function has the most important task, it means to make the script code as synoptic as possible. The basic thought is to distinguish particular code parts as described below :

a) keywords, such as scripting commands and functions (in cooperation with Quick Keywords Menu can be the first keywords' letter inserted in Upper-case); default color Blue,
b) comments; default color Green,
c) double-quoted text; default color Light Violet,
d) errors (in cooperation with the Code Checker the error description (if any) will be shown in the bottom panel); default underline color Red,
e) in addition after the code is cheked against any error important code info will be shown in the right panel. What kind of info depends on a file type. For SQS it is a Labels list, for EXT, SQM, CPP it is a classes structure and hierarchy including constants definitions, for CSV it is a string sets names list (when you double-click on an item in this info the caret will be moved at an appropriate position).

Colors described above can be changed any time in the Options window (menu Tools -> Options).

Syntax highlighting works as follows :
1) After a file is opened a check against errors is performed and then the file content is shown,
2) During the file editing all operations described above are performed each time the caret leaves its curent line, it doesn't matter why.
